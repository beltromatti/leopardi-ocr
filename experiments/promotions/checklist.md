# Promotion Checklist

An experiment can be promoted only if:

1. the experiment exists in `registry/experiment-index.csv`
2. the config stack is fully declared
3. the dataset version is declared
4. the benchmark protocol version is declared
5. the relevant public and internal benchmarks were run
6. the latency card is present
7. failure slices were inspected
8. no data leakage issue is open
9. the promoted track being replaced is identified explicitly
